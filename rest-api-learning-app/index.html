What is a REST API?

Simple Explanation

Think of a REST API as a waiter in a restaurant:

  • You (Client): The customer who wants something
  • API (Waiter): Takes your request to the kitchen
  • Server (Kitchen): Prepares what you requested
  • Response: The waiter brings you your food

Technical Definition

REST stands for REpresentational State Transfer.

It's a way for computers to talk to each other over the internet using standard HTTP methods.

Real-World Example

When you check the weather on your phone:

Your App ‚Üí (Request) ‚Üí Weather API
Weather API ‚Üí (Response with data) ‚Üí Your App
Your App ‚Üí Displays "72¬∞F, Sunny"

Key REST Principles

  1. Client-Server: Your script (client) talks to a server
  2. Stateless: Each request is independent (server doesn't remember you)
  3. Uniform Interface: All APIs work similarly (GET, POST, etc.)
  4. Resources: Everything is a "resource" with a URL (like /users, /posts)

HTTP Methods (Verbs)

HTTP methods tell the API what you want to do. Think of them as action words:

GET

Retrieve/Read Data

Like asking "Can I see the menu?"

GET /api/users/123

Get information about user #123

POST

Create New Data

Like saying "I'd like to place an order"

POST /api/users

Create a new user

PUT

Update/Replace Data

Like saying "Change my entire order"

PUT /api/users/123

Replace all data for user #123

PATCH

Partially Update Data

Like saying "Just add extra cheese"

PATCH /api/users/123

Update only specific fields of user #123

DELETE

Remove Data

Like saying "Cancel my order"

DELETE /api/users/123

Delete user #123

Understanding Requests & Responses

üì§ Request (What You Send)

Request Components:

  1. URL: Where to send the request
  2. Method: What action (GET, POST, etc.)
  3. Headers: Metadata (like content type, auth)
  4. Body: Data you're sending (for POST/PUT)

Example Request:

GET /api/users/123 HTTP/1.1
Host: example.com
Authorization: Bearer abc123
Content-Type: application/json

üì• Response (What You Get Back)

Response Components:

  1. Status Code: Success or error (200, 404, etc.)
  2. Headers: Metadata about the response
  3. Body: The actual data (usually JSON)

Example Response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 123,
  "name": "John Doe",
  "email": "john@example.com"
}

Real-World Flow

// 1. You want user data
Request: GET https://api.example.com/users/123

// 2. Server processes request

// 3. Server sends response
Response:
{
  "status": 200,
  "data": {
    "id": 123,
    "name": "John Doe",
    "email": "john@example.com",
    "joined": "2024-01-15"
  }
}

// 4. Your script uses the data
console.log(data.name); // "John Doe"

HTTP Status Codes

Status codes tell you if your request succeeded or failed. They're like traffic lights:

2xx - Success ‚úÖ

  • 200 OK: Request succeeded, here's your data
  • 201 Created: Successfully created new resource
  • 204 No Content: Success, but no data to return

3xx - Redirection ‚Ü™Ô∏è

  • 301 Moved Permanently: Resource moved to new URL
  • 304 Not Modified: Cached version is still valid

4xx - Client Errors ‚ö†Ô∏è

  • 400 Bad Request: Your request is malformed
  • 401 Unauthorized: You need to log in
  • 403 Forbidden: You don't have permission
  • 404 Not Found: Resource doesn't exist
  • 429 Too Many Requests: Slow down!

5xx - Server Errors üî•

  • 500 Internal Server Error: Server crashed
  • 503 Service Unavailable: Server is down/overloaded

How to Handle Status Codes in Code

// Check status code in response
if (response.status === 200) {
    console.log("Success!", response.data);
} else if (response.status === 404) {
    console.log("Not found - check your URL");
} else if (response.status === 401) {
    console.log("Need to authenticate first");
} else if (response.status >= 500) {
    console.log("Server error - try again later");
}

Headers & Authentication

What are Headers?

Headers are metadata about your request/response. Think of them as the envelope of a letter - they contain information about the contents.

Common Request Headers

Content-Type

Tells server what format you're sending

Content-Type: application/json

Accept

Tells server what format you want back

Accept: application/json

Authorization

Proves who you are (credentials)

Authorization: Bearer YOUR_TOKEN

User-Agent

Identifies your application

User-Agent: MyScript/1.0

Authentication Methods

1. API Keys

Simple string token

// In URL
GET /api/data?api_key=abc123

// In Header
headers: {
  'X-API-Key': 'abc123'
}

2. Bearer Tokens

Most common for modern APIs

headers: {
  'Authorization': 'Bearer eyJhbGc...'
}

3. Basic Auth

Username and password

// Encoded as base64
headers: {
  'Authorization': 'Basic dXNlcjpwYXNz'
}

‚ö†Ô∏è Security Tips

  • Never share API keys publicly
  • Use HTTPS (not HTTP) for secure transmission
  • Rotate API keys regularly
  • Don't commit API keys to GitHub!

Data Formats: JSON & XML

JSON (JavaScript Object Notation)

Most popular format - easy to read and use

JSON Example:

{
  "user": {
    "id": 123,
    "name": "John Doe",
    "email": "john@example.com",
    "active": true,
    "tags": ["developer", "beginner"],
    "address": {
      "city": "New York",
      "country": "USA"
    }
  }
}

Working with JSON:

// Parse JSON string to object
const obj = JSON.parse(jsonString);
console.log(obj.user.name); // "John Doe"

// Convert object to JSON string
const json = JSON.stringify(obj);
console.log(json);

XML (eXtensible Markup Language)

Older format - still used by some APIs

XML Example:

<user>
  <id>123</id>
  <name>John Doe</name>
  <email>john@example.com</email>
  <active>true</active>
  <tags>
    <tag>developer</tag>
    <tag>beginner</tag>
  </tags>
</user>

Why JSON is Preferred

  • Lighter weight (less data to transfer)
  • Native to JavaScript (easy to use)
  • Easier for humans to read
  • Faster to parse

Introduction to Greasemonkey/Tampermonkey Scripts

What are Userscripts?

Userscripts (Greasemonkey/Tampermonkey scripts) are JavaScript programs that run in your browser to modify web pages or add new features.

What Can You Do?

üé® Modify Appearance

Change colors, layout, remove ads

üìä Add Data

Fetch and display data from APIs

‚ö° Automate

Auto-fill forms, click buttons

üîß Enhance

Add new features to websites

Basic Script Structure

// ==UserScript==
// @name         My First Script
// @namespace    http://tampermonkey.net/
// @version      1.0
// @description  Enhances the webpage
// @author       You
// @match        https://example.com/*
// @grant        GM_xmlhttpRequest
// ==/UserScript==

(function() {
    'use strict';

    // Your code here
    console.log('Script is running!');

})();

Metadata Block Explained

@name - Name of your script
@version - Version number
@description - What your script does
@match - Which websites to run on
@grant - Special permissions needed

Simple Example: Add Text to Page

// ==UserScript==
// @name         Hello World
// @match        https://*/*
// @grant        none
// ==/UserScript==

(function() {
    'use strict';

    // Create a div element
    const div = document.createElement('div');
    div.textContent = 'Hello from Greasemonkey!';
    div.style.cssText = 'position:fixed; top:10px; right:10px; background:yellow; padding:10px; z-index:9999;';

    // Add it to the page
    document.body.appendChild(div);
})();

Making API Calls in Greasemonkey Scripts

Why Use GM_xmlhttpRequest?

Regular fetch() or XMLHttpRequest are blocked by CORS (Cross-Origin Resource Sharing). Greasemonkey's GM_xmlhttpRequest bypasses this restriction.

Basic Syntax

// Remember to add @grant GM_xmlhttpRequest in metadata!

GM_xmlhttpRequest({
    method: "GET",
    url: "https://api.example.com/data",
    headers: {
        "Content-Type": "application/json"
    },
    onload: function(response) {
        // Success! Handle the response
        console.log(response.responseText);
    },
    onerror: function(error) {
        // Handle errors
        console.error("Request failed:", error);
    }
});

GET Request Example

Fetch Weather Data

// ==UserScript==
// @name         Weather Display
// @match        https://*/*
// @grant        GM_xmlhttpRequest
// @connect      api.openweathermap.org
// ==/UserScript==

(function() {
    'use strict';

    const API_KEY = 'your_api_key_here';
    const CITY = 'London';

    GM_xmlhttpRequest({
        method: "GET",
        url: `https://api.openweathermap.org/data/2.5/weather?q=${CITY}&appid=${API_KEY}&units=metric`,
        onload: function(response) {
            const data = JSON.parse(response.responseText);

            // Create display element
            const weatherDiv = document.createElement('div');
            weatherDiv.innerHTML = `
                Weather in ${data.name}:

                Temperature: ${data.main.temp}¬∞C

                Conditions: ${data.weather[0].description}
            `;
            weatherDiv.style.cssText = 'position:fixed; top:10px; right:10px; background:lightblue; padding:15px; border-radius:5px; z-index:9999;';

            document.body.appendChild(weatherDiv);
        },
        onerror: function(error) {
            console.error("Weather fetch failed:", error);
        }
    });
})();

POST Request Example

Send Data to API

GM_xmlhttpRequest({
    method: "POST",
    url: "https://api.example.com/users",
    headers: {
        "Content-Type": "application/json",
        "Authorization": "Bearer YOUR_TOKEN"
    },
    data: JSON.stringify({
        name: "John Doe",
        email: "john@example.com"
    }),
    onload: function(response) {
        if (response.status === 201) {
            console.log("User created!");
            const newUser = JSON.parse(response.responseText);
            console.log("New user ID:", newUser.id);
        }
    }
});

Important Parameters

method

"GET", "POST", "PUT", "DELETE", etc.

url

API endpoint to call

headers

Object with header key-value pairs

data

Body content (for POST/PUT)

onload

Function called on success

onerror

Function called on failure

Don't Forget @connect!

Add @connect directive for each domain you'll call:

// @connect api.example.com

Exercise 1: Random Quote Fetcher

Goal: Create a script that fetches a random quote and displays it on any webpage.

API: https://api.quotable.io/random

Requirements:

  • Use GM_xmlhttpRequest to fetch the quote
  • Display the quote and author in a styled div
  • Position it in the bottom-right corner
  • Add a button to fetch a new quote
// ==UserScript==
// @name         Random Quote Display
// @match        https://*/*
// @grant        GM_xmlhttpRequest
// @connect      api.quotable.io
// ==/UserScript==

(function() {
    'use strict';

    // Create container
    const container = document.createElement('div');
    container.style.cssText = 'position:fixed; bottom:20px; right:20px; background:white; padding:20px; border-radius:10px; box-shadow:0 4px 6px rgba(0,0,0,0.1); max-width:300px; z-index:9999;';

    const quoteText = document.createElement('p');
    quoteText.style.cssText = 'margin:0 0 10px 0; font-style:italic;';

    const authorText = document.createElement('p');
    authorText.style.cssText = 'margin:0 0 10px 0; font-weight:bold; text-align:right;';

    const refreshBtn = document.createElement('button');
    refreshBtn.textContent = 'New Quote';
    refreshBtn.style.cssText = 'padding:8px 16px; background:#007bff; color:white; border:none; border-radius:5px; cursor:pointer;';

    container.appendChild(quoteText);
    container.appendChild(authorText);
    container.appendChild(refreshBtn);
    document.body.appendChild(container);

    // Function to fetch quote
    function fetchQuote() {
        GM_xmlhttpRequest({
            method: "GET",
            url: "https://api.quotable.io/random",
            onload: function(response) {
                const data = JSON.parse(response.responseText);
                quoteText.textContent = `"${data.content}"`;
                authorText.textContent = `- ${data.author}`;
            },
            onerror: function(error) {
                quoteText.textContent = "Failed to load quote";
                console.error(error);
            }
        });
    }

    // Initial fetch
    fetchQuote();

    // Fetch new quote on button click
    refreshBtn.addEventListener('click', fetchQuote);
})();

Exercise 2: GitHub User Info

Goal: Create a script that shows GitHub user information.

API: https://api.github.com/users/{username}

Requirements:

  • Add an input field to enter a GitHub username
  • Fetch user data when button is clicked
  • Display: avatar, name, bio, public repos count
  • Handle errors (user not found)
// ==UserScript==
// @name         GitHub User Info
// @match        https://*/*
// @grant        GM_xmlhttpRequest
// @connect      api.github.com
// ==/UserScript==

(function() {
    'use strict';

    // Create UI
    const container = document.createElement('div');
    container.style.cssText = 'position:fixed; top:20px; right:20px; background:white; padding:20px; border-radius:10px; box-shadow:0 4px 6px rgba(0,0,0,0.1); min-width:250px; z-index:9999;';

    const input = document.createElement('input');
    input.placeholder = 'GitHub username';
    input.style.cssText = 'width:100%; padding:8px; margin-bottom:10px; border:1px solid #ddd; border-radius:4px;';

    const searchBtn = document.createElement('button');
    searchBtn.textContent = 'Search';
    searchBtn.style.cssText = 'padding:8px 16px; background:#28a745; color:white; border:none; border-radius:5px; cursor:pointer; width:100%;';

    const resultDiv = document.createElement('div');
    resultDiv.style.cssText = 'margin-top:15px;';

    container.appendChild(input);
    container.appendChild(searchBtn);
    container.appendChild(resultDiv);
    document.body.appendChild(container);

    // Search function
    searchBtn.addEventListener('click', function() {
        const username = input.value.trim();
        if (!username) {
            resultDiv.innerHTML = '
Please enter a username
';
            return;
        }

        resultDiv.innerHTML = '
Loading...
';

        GM_xmlhttpRequest({
            method: "GET",
            url: `https://api.github.com/users/${username}`,
            onload: function(response) {
                if (response.status === 200) {
                    const user = JSON.parse(response.responseText);
                    resultDiv.innerHTML = `
                        
                        
${user.name || user.login}

                        
Bio: ${user.bio || 'N/A'}

                        
Public Repos: ${user.public_repos}

                        
Followers: ${user.followers}

                        View Profile
                    `;
                } else if (response.status === 404) {
                    resultDiv.innerHTML = '
User not found!
';
                } else {
                    resultDiv.innerHTML = '
Error loading user
';
                }
            },
            onerror: function() {
                resultDiv.innerHTML = '
Network error
';
            }
        });
    });
})();
